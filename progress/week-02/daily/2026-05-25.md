## 📖 Concepts Studied

### PBR — Physically Based Rendering

A rendering model that simulates real-world light behavior mathematically. The key advantage over legacy shading models is consistency: a PBR material looks correct under any lighting condition without requiring manual per-light adjustment. UE5's material system is built on PBR by default.

**The three primary properties:**

| Property | Range | Role |
| --------- | ----------- | ----------------------------------------------- |
| Base Color | RGB (0–1) | Diffuse albedo of the surface |
| Roughness | 0.0 → 1.0 | Controls spread of specular reflection |
| Metallic | 0 or 1 | Toggles between dielectric and conductor model |

---

### Normal Map

A texture that encodes surface orientation data — not color — to simulate geometric detail without adding polygons. Each pixel stores a direction vector encoded as RGB:

- **R (Red) = X axis**
- **G (Green) = Y axis**
- **B (Blue) = Z axis**

The characteristic purple/blue color of a normal map is the visual result of a flat surface pointing straight up: `(0.5, 0.5, 1.0)` in normalized RGB, which decodes to `(0, 0, 1)` in world-space XYZ.

**TA note:** Normal maps cannot be authored manually in a paint application and must be generated from high-poly geometry in software such as Blender or Substance Painter. They are one of the highest value-to-cost techniques in real-time rendering — significant visual detail for zero additional draw calls.

---

### Specular

Controls the Fresnel reflectivity of non-metallic surfaces. Default value is `0.5`, which maps to approximately 4% reflectivity at normal incidence — accurate for most real-world dielectrics including skin, stone, wood, and fabric.

**Practical rules:**
- Leave at `0.5` for the vast majority of materials.
- Lower only for intentionally non-reflective micro-cavities such as cracks, pores, or fabric gaps.
- **Specular is completely ignored when Metallic = 1.** The Fresnel response for conductors is driven by Base Color instead.

---

### Roughness

Defines the microscopic surface irregularity that scatters specular reflections.

| Value | Visual Result | Real-world analogy |
| ----- | ------------------------- | --------------------------------- |
| 0.0 (black) | Mirror-sharp reflection | Polished glass, still water |
| 0.5 | Soft, diffused reflection | Brushed metal, worn plastic |
| 1.0 (white) | Fully matte, no highlight | Raw concrete, unfinished wood |

Roughness maps are an artistic decision that conveys the surface history — wear patterns, dirt accumulation, polished contact areas. They carry storytelling information that is not recoverable once removed.

**TA note:** Roughness maps must never be stripped during optimization without explicit sign-off from the responsible artist. Their removal changes the material's narrative intent, not just its technical budget.

---

### Metallic

Switches the material between two fundamentally different light interaction models:

- **Metallic = 0 (dielectric):** Surface has both diffuse and specular components. Base Color represents the albedo seen in diffuse lighting.
- **Metallic = 1 (conductor):** Diffuse component is eliminated entirely. Base Color becomes the tinted specular reflectance color.

In practice, metallic values should be either `0` or `1`. Intermediate values (0.1–0.9) have no physical analog and are valid only for transitional blending in masks.

**No real-world metal has a dark Base Color.** All conductors have a minimum reflectance of approximately 180 sRGB. Reference values:

| Material | R | G | B |
| ---------- | --- | --- | --- |
| Chromium | 195 | 195 | 195 |
| Platinum | 213 | 208 | 200 |
| Iron | 196 | 199 | 199 |
| Aluminium | 245 | 245 | 247 |
| Copper | 250 | 209 | 194 |
| Silver | 250 | 247 | 242 |
| Gold | 255 | 219 | 145 |

### Node Graph

| Step | Node | Purpose |
| ---- | -------------------------------- | --------------------------------------------------- |
| 1 | **TextureCoordinate** | Controls UV scale via UTiling / VTiling parameters |
| 2 | **TextureSample** × 3 | Samples: Base Color, Normal, Roughness textures |

## 🔍 Technical Findings

### UE5.4 Base Pass Shader Instruction Count

The Stats panel reports approximately **222 instructions** for the base pass shader on this material, compared to roughly **87 instructions** in a comparable UE4 setup. This is expected and is not a configuration error.

**Root cause:** UE5.4 enables **Lumen** (dynamic global illumination) and **Nanite** (virtualized geometry) by default. Both systems add shader complexity to every material's base pass as part of their evaluation pipeline. This is the performance baseline for UE5.4, not an outlier.

### Texture Sampler Budget

Current session used **6 of 16** available texture samplers. The 16-sampler limit is a hardware constraint relevant to material complexity optimization in Phase 3.

## 📌 TA-Relevant Notes

- **Normal maps are a zero-cost visual upgrade** — they add apparent surface detail with no polygon overhead. In Phase 3, they should be preserved (or introduced) rather than removed when optimizing.
- **Roughness maps carry narrative information.** Stripping them does not just reduce a shader instruction — it removes intentional artistic decisions about surface history. Always consult the responsible artist before touching roughness in optimization passes.
- **Metallic = 1 fundamentally changes GPU light evaluation.** It is not simply "making something shiny." The underlying shading path changes, which affects how Base Color data is interpreted at the hardware level.
- **TextureCoordinate is a dependency node**, not optional. All TextureSample nodes in a material must share the same UV source to remain visually coherent when tiling parameters change.
